1Pick a city & lead time
→
Scroll the city tabs at the top to pick one of the 20 markets. Lead time buttons (Today / +1d / +2d / +3d / +5d) control which day's contracts you're analyzing.
→
Today: tightest error bars (~1.2–1.8°F). Best for intraday Bayesian signals. +3d/+5d: wider error bars but more mispricing opportunities — fewer traders focus out here.
2Reading the model bar chart
◼
Colored bar = that model's forecast for the daily high. Longer = higher predicted temp.
◻
Shaded region = ±1σ historical error for that lead time. Wider = less reliable. MOS has the tightest bars; NAM at +5d the widest.
⏐
Red vertical line = Kalshi market mid converted to °F. When model bars consistently fall to one side of the red line, the market may be mispriced.
3Reading the probability curves
→
Each bell curve = one model's full probability distribution over possible high temps. Peak = most likely outcome. Width = uncertainty.
→
Tight cluster of curves = model agreement = higher conviction. Spread-out curves = disagreement = reduce position size or pass.
4Reading the edge scanner
Strike
Each row = one Kalshi YES contract: "Will the daily high be ≥ X°F?"
Mkt%
Implied probability from the Kalshi contract price. ~50% = coin flip, ~90% = market expects it to happen.
Mdl%
What the weighted model blend says the true probability is, using each model's historical RMSE as the uncertainty weight.
Edge
Model% − Market%. Green = buy YES (models think it's more likely than priced). Red = buy NO. Signals beyond ±8% are potentially actionable after Kalshi's ~4% fee.
5Intraday Bayesian update
→
Once the day starts, current ASOS observations dramatically improve accuracy. We update the model prior with current obs + a seasonal afternoon heating rate to project the likely daily high.
→
The Δ column shows how much each strike's probability shifted vs. the model-only prior. Large Δ with a tight ensemble = the market likely hasn't repriced yet — that's your window.
→
By noon local time, posterior σ typically falls to ~0.8–1°F. The most actionable intraday signals appear here.
📡 Data & settlement sources
◆
Open-Meteo (free, no key) — GFS, ECMWF, NAM, ICON, GEM model forecasts, updated every 6h run cycle.
◆
ASOS observations via Open-Meteo past-hours endpoint — same station network Kalshi contracts settle against.
◆
Settlement: all contracts settle on the NWS Daily Climate Report, released the following morning. Note: NWS uses local standard time, so during Daylight Saving Time the obs window is 1:00 AM–12:59 AM local the next day.
◆
Kalshi market mid is estimated from model consensus. Add your API key to the fetchKalshiMid() function for live contract prices.
